#  1. Propósito da Avaliação

A presente avaliação tem como objetivo analisar a qualidade do sistema Agio uma aplicação web open source de gestão de inventário sob as perspectivas de:

<div class="grid cards" markdown>

- **1. Adequação Funcional**
- **2. Confiabilidade**
- **3. Eficiência de Desempenho**

</div>

As características selecionadas seguem o modelo de qualidade definido pela norma **ISO/IEC 25010:2011 (SQuaRE)**.

---

## Questões de Decisão

A avaliação busca responder às seguintes questões:

| Questão | Tomador de Decisão | Uso Pretendido |
|:--|:--|:--|
| As funcionalidades implementadas atendem ao backlog planejado? | Equipe de desenvolvimento e mantenedores | Priorizar correções e futuras implementações |
| O sistema mantém estabilidade sob uso contínuo e entradas inesperadas? | Operadores e administradores | Verificar prontidão para ambiente de produção |
| O desempenho é aceitável para empresas de pequeno/médio porte? | Potenciais adotantes do software | Avaliar viabilidade de adoção do sistema |

---

!!! info "Resultado Esperado"

    Os resultados da avaliação serão documentados em um relatório técnico
    com dados auditáveis, disponibilizado no repositório Git do projeto.

    As informações poderão auxiliar futuras evoluções do sistema e orientar
    a comunidade open source sobre a qualidade do software avaliado.
